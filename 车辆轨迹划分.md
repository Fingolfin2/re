# 车辆轨迹划分工作流（来自两份 Notebook）

来源：
- `re/AdaptiveTimeThreshold.ipynb`
- `re/TrajectoryExtract.ipynb`

## 1. 总体流程

1. 读取预处理后的过车数据（`center_valid_df.csv`）。
2. 统计全样本相邻卡口记录时间间隔，输出 `全样本相邻卡口记录时间间隔.csv`。
3. 基于 IsolationForest，为每个相邻卡口对构建“自适应行程时间阈值表”，输出 `kkod_time_threshold.npy`。
4. 计算固定时间阈值 `B`（来自全样本时间间隔的分位数）。
5. 读取卡口到路网节点映射（`center_realpos.npy`），得到 `KKBH -> 路网节点` 映射。
6. 逐车遍历过车序列，按“固定阈值 + 自适应阈值”切分单次出行轨迹，输出多份 `path_*.npy`。

## 2. 主要输入与字段

### 2.1 过车数据（CSV）

- 文件：`center_valid_df.csv`（Notebook 中使用 `encoding='gbk'`）
- Notebook 依赖字段：
  - `CLTMBH`：车辆标识/号牌
  - `GCSJ`：过车时间（需要解析为 `datetime`）
  - `KKBH`：卡口编号（需要转 `str`）
  - `GCRQ`：过车日期（用于约束“同一天”）

注意：AdaptiveTimeThreshold 里用“相邻两行”统计相邻卡口时间间隔，隐含强假设：数据已按 `(CLTMBH, GCRQ, GCSJ)` 排序。

### 2.2 卡口到路网节点映射（NPY）

- 文件：`center_realpos.npy`
- Notebook 用法：
  - `realpos[0]`：路网节点序列（`realpos_name`）
  - `realpos[1]`：卡口编号序列（`bayonet_name`）
  - 通过 `dict(zip(bayonet_name, realpos_name))` 构建 `bayonet_to_pos`

## 3. Step A：全样本相邻卡口记录时间间隔（AdaptiveTimeThreshold）

### 3.1 生成规则

顺序扫描 `df` 的相邻两行 `i, i+1`：
- 若 `df[i].CLTMBH == df[i+1].CLTMBH` 且 `df[i].GCRQ == df[i+1].GCRQ`：
  - `KKOD = KKBH[i] + '-' + KKBH[i+1]`
  - `time = (GCSJ[i+1] - GCSJ[i]).total_seconds()`
  - `hour = GCSJ[i].hour`

汇总为 `TravelTime(KKOD, time, hour)` 并保存：
- 输出：`全样本相邻卡口记录时间间隔.csv`

## 4. Step B：构建自适应阈值表（AdaptiveTimeThreshold）

### 4.1 时段划分

- 高峰（high）：`[7, 8, 17, 18]`
- 平峰（flat）：`[9,10,11,12,13,14,15,16,19,20,21]`
- 低峰（low）：`[0,1,2,3,4,5,6,22,23]`

### 4.2 阈值生成逻辑

对每个相邻卡口对 `KKOD`（记为 `kkod_name[i]`）：

1. 若 `std(time) < 100` 或样本量 `< 50`：
   - 不做异常值滤除
   - 仅写入一条记录：`period='all'`，阈值为 `(min(time), median(time), max(time))`
2. 否则尝试按 high/flat/low 分段：
   - 若三段中任意一段样本为空：退化为 `period='all'`（同上）
   - 否则对每段：
     - 训练 `IsolationForest(n_estimators=15, max_features=1, contamination='auto', random_state=2023)`
     - 过滤异常：保留 `anomaly == 1` 的样本
     - 写入该段的阈值：`(min(time), median(time), max(time))`

最终保存：
- 输出：`kkod_time_threshold.npy`
- 每行结构：`[kkod, period, min, median, max]`

## 5. Step C：固定时间阈值 B（TrajectoryExtract）

从 `全样本相邻卡口记录时间间隔.csv` 读取 `time` 列，计算：
- `B = quantile(time, 0.70)`

并在后续使用：
- 默认最大阈值 `round(1.5 * B)`
- 同时对自适应阈值的上限做裁剪：`maxtime = min(maxtime, round(1.5 * B))`

## 6. Step D：单次出行轨迹切分（TrajectoryExtract）

### 6.1 需要的预加载

- 读取 `kkod_time_threshold.npy` 并构建查询结构：
  - 原版 Notebook：`kkod_time_df` DataFrame 后再过滤
  - 优化版 Notebook：构建 `kkod_map[kkod][period] = (min, max)`
- 构建 `bayonet_to_pos`（卡口 -> 路网节点）
- `vehicle_list = df['CLTMBH'].unique()`（或去重列表）

### 6.2 合理时间阈值区间查询（每一对相邻卡口）

对相邻卡口对：
- `kkod = prev_kkbh + '-' + curr_kkbh`
- `hour = 当前过车时间.hour`，得到 `period ∈ {high, flat, low}`

查询阈值 `[mintime, maxtime]`：
- 若 `kkod` 不存在：`mintime = 1`，`maxtime = round(1.5*B)`
- 若存在但只有 `period='all'`：使用该 `(min,max)`，但：
  - 若 min 为 nan，或 `min >= B 且 (max - min) < 60`：回退到默认区间
- 若存在且区分时段：按 `hour` 取 high/flat/low 对应 `(min,max)`，并应用同样的 nan/回退逻辑
- 最终：`maxtime = min(maxtime, round(1.5*B))`

### 6.3 切分规则（Notebook 原版实现的语义）

对单车过车序列，维护当前“单次出行”容器 `st_*`（pos/bayonet/time）：

- 若前后卡口不同：
  - 若 `mintime <= ti <= maxtime` 且 `ti != 0`：归为同一次出行，追加当前点
  - 若 `ti > maxtime`：结束当前出行并开启新出行（当前段要求点数>1 且总时长>300s 才保存）
  - 否则（`ti < mintime` 或 `ti == 0`）：丢弃当前点
- 若前后卡口相同：
  - 若 `ti > B`：结束当前出行并开启新出行（同样有点数>1、总时长>300s 的保存条件）
  - 否则：丢弃当前点

其中 `ti` 在原版 Notebook 中的计算口径是：
- `ti = 当前点时间 - 当前单次出行最后一次保留点时间`

### 6.4 优化版（并行）实现差异（TrajectoryExtract 最后一个单元）

Notebook 末尾给了一版 `joblib.Parallel(n_jobs=-1)` 的实现，其关键差异点：
- `ti` 用的是严格相邻记录差：`ti = row['GCSJ'] - row['prev_time']`
- 触发“继续/切分”的条件与原版逐点容器维护方式不完全一致

后续重构成 `.py` 时，需要明确以哪一版逻辑为准（以保证语义一致）。

## 7. 输出产物（TrajectoryExtract）

主输出（与 Notebook 保持一致的 `.npy` 结构）：
- `path_pos.npy`：每次出行经过的路网节点序列（list[list[node]])
- `path_bayonet.npy`：每次出行经过的卡口序列（list[list[kkbh]])
- `path_time.npy`：每次出行对应的过车时间序列（list[list[datetime]])
- `path_start_time.npy`：每次出行起始时间（list[datetime]）
- `path_stop_time.npy`：每次出行结束时间（list[datetime]）
- `path_travel.npy`：每次出行总时长（秒）（list[float/int]）
- `path_cltmbh.npy`：每次出行对应车辆标识（list[str]）

可选统计输出：
- `path_bayonet_number.npy`：每次出行的卡口记录数（list[int]）
